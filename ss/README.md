# Screenshots for the Final Project PPT

Drop your screenshots in this folder. Use these exact filenames so the
presentation can pick them up automatically.

| # | Filename                | What to capture                                                                 |
| - | ----------------------- | ------------------------------------------------------------------------------- |
| 1 | `01-jira-board.png`     | Jira board view showing the project columns and tickets.                        |
| 2 | `02-jira-ticket.png`    | One open Jira ticket with description and status.                               |
| 3 | `03-git-history.png`    | `git log --oneline --graph --all` or GitHub/GitLab commits page.                |
| 4 | `04-folder-structure.png` | VS Code sidebar showing `src/{domain, application, infrastructure, interfaces}`. |
| 5 | `05-server-startup.png` | Terminal running `npm run dev` with Winston "Server running on port 3000".      |
| 6 | `06-prisma-studio.png`  | `npx prisma studio` showing users / spaces / bookings tables with seeded rows.  |
| 7 | `07-bruno-login.png`    | Bruno running "Login Admin" with the JWT token in the response.                 |
| 8 | `08-bruno-create-booking.png` | Bruno running "Create Booking (User)" with a 201 response.                |
| 9 | `09-bruno-admin-confirm.png` | Bruno running "Confirm Booking (Admin)" with a 200 response.               |
|10 | `10-bruno-403.png`      | Bruno running "Create Space Forbidden (403)" showing the 403 response.          |
|11 | `11-bruno-availability.png` | Bruno running "Check Availability" with the availability payload.           |
|12 | `12-test-output.png`    | Terminal output of `npm test` showing pass/fail summary.                        |
|13 | `13-coverage-report.png` | Jest coverage summary (statements/branches/functions/lines %).                 |
|14 | `14-winston-log-file.png` | `logs/combined.log` open in the editor showing request logs.                  |

Tips:
- PNG is best for crisp UI screenshots.
- Landscape / wide crops work best on slides.
- Keep sensitive info (real emails, real tokens) out of the frame if possible.
